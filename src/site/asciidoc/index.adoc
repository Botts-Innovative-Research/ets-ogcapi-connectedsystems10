= OGC API - Connected Systems Conformance Test Suite

== Scope

This executable test suite (ETS) verifies implemented portions of
OGC API - Connected Systems Part 1: Feature Resources (OGC 23-001) and
implemented portions of OGC API - Connected Systems Part 2: Dynamic Data
(OGC 23-002). Conformance testing is black-box testing of the externally
visible HTTP behavior of the implementation under test.

.Relevant specifications
[caption="Figure 1: "]
image::img/specifications.png[]

TeamEngine 6 is the forward runtime for this package. Local OSH is the primary
development E2E target; GeoRobotix is available only as an explicit advisory
interoperability probe.

== Implemented coverage

The suite includes Part 1 Core/Common/System resource coverage, additional
implemented Part 1 conformance-class subsets, and implemented Part 2 dynamic-data
subsets for datastreams, control streams, feasibility, events, encodings, and
binding closure. Some Part 2 checks are declaration-gated or require a populated
local OSH IUT for positive evidence.

== Test suite structure

The test suite definition file (testng.xml) is located in the root package,
`org.opengis.cite.ogcapiconnectedsystems10`. TestNG groups model the dependency
graph between conformance-class subsets, and tests skip honestly when an IUT does
not declare the corresponding conformance class or lacks required data.

|Value domain
|Obligation
|Description

|String
|O
|Opaque Authorization header value used for authenticated local runs.

|Boolean
|O
