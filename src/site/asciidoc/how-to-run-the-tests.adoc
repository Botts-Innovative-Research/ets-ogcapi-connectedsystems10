== How to run the tests

The entry point is the CS API landing page for the implementation under test.
TeamEngine 6 is the supported runtime path for this OGC API - Connected Systems
ETS. Current coverage is partial for Part 1 (OGC 23-001) and Part 2 (OGC
23-002).

=== 1. Integrated development environment

Set the main class to:

`org.opengis.cite.ogcapiconnectedsystems10.TestNGController`

The first argument is an XML properties file containing the test run arguments.
If it is not supplied, the controller uses `${user.home}/test-run-props.xml`.

[source,xml]
----
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE properties SYSTEM "http://java.sun.com/dtd/properties.dtd">
<properties version="1.0">
  <comment>OGC API - Connected Systems partial Part 1 and Part 2 test run arguments</comment>
  <entry key="iut">http://field-hub-osh-1:8081/sensorhub/api</entry>
  <entry key="auth-credential"></entry>
  <entry key="mutation-tests-enabled">false</entry>
  <entry key="mutation-iut-policy"></entry>
</properties>
----

=== 2. Command shell

The all-in-one JAR can run the same properties file from a shell:

`java -jar ets-ogcapi-connectedsystems10-0.1-SNAPSHOT-aio.jar [-o|--outputDir $TMPDIR] [test-run-props.xml]`

=== 3. Docker and TeamEngine 6

The pinned OGC TeamEngine 6 digest is linux/amd64-only. On a linux/arm64 Docker
host, register amd64 binfmt support before starting the container:

[source,bash]
----
docker run --privileged --rm tonistiigi/binfmt --install amd64
----

[source,bash]
----
docker compose up --build
----

The TeamEngine UI is served at `http://localhost:8081/teamengine/`.

=== 4. Local OSH smoke gate

[source,bash]
----
SMOKE_DOCKER_NETWORK=field-hub_default \
  SMOKE_IUT_URL=http://field-hub-osh-1:8081/sensorhub/api \
  SMOKE_AUTH_CREDENTIAL="$SMOKE_AUTH_CREDENTIAL" \
  SMOKE_OUTPUT_DIR=/tmp/ets-ogcapi-connectedsystems10-local-osh-results \
  bash scripts/smoke-test.sh
----

Use GeoRobotix only when intentionally running an advisory public
interoperability probe.
